==> File permissions using numbers;
0 - no permissions
1 - execute
2 - write
3 - 2+1 --- write+execute
4 - read
5 - 4+1 --- read + execute
6 - 4+2 --- read + write
7 - 6+1 --- read + write + execute

ugo + x
111

ugo + w
222

u+r g+rx o+rw
456

u-rwx g-rwx o+rwx
007 (or) 7

==> User management
ubuntu (or) ec2-user
root

To see the list of users available
cat /etc/passwd

To create users
sudo useradd <username>

To switch to the custom user
su kastro

To rename the user
usermod -l kastro-devops kastro

To see user got created or not
id kastro-devops

To see the list of groups
cat /etc/group

To create a group
sudo groupadd devops-team

To add an user to a group
sudo usermod -aG devops-team kastro-devops

To remove the user from a group
sudo gpasswd -d kiran devops-team

To see the list of users available in a group
sudo lid -g devops-team

To delete a group 
sudo groupdel devops-team

To rename a group
sudo groupmod -n kastro-devops-group kastro

==> Understanding sudoers file
Using this file we can control which user can run what command(s)

To print the data available in sudoers file
 sudo cat /etc/sudoers

To open/edit the sudoers file
sudo visudo

==> Modifying the owners and group for a file/directory
CHOWN Command

To change the ownership of a file
sudo chown kiran linxux.txt

To change group of a file
sudo chown :kiran linxux.txt

==> SYSTEMCTL

	1. start a service
	2. stop a service
	3. restart a service
	4. enable a service
	5. reload a service
